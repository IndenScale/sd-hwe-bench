# 6. 评估

## 6.1 研究问题

- **RQ1.** ADL 能否以文本原生、可被机器检查的形式表达真实工程设计？
- **RQ2.** ESA 能否在设计时检测常见违规，延迟满足 CI/CD 要求且信噪比高于几何 ACC？
- **RQ3.** 该表示是否提供 RLVR Agent 训练所需的可验证奖励信号？

评估基于 piki 样本和受控违规注入实验，测量表达力、检查延迟、规则覆盖、检测准确率和交付物生成（笔记本，热缓存，五次中位数）。

## 6.3 违规注入实验

在电信样本 ADL 中注入 15 个已知违规：

| 类别 | 数量 | 示例 |
|------|-----|------|
| L2：接口不匹配 | 4 | SFP28 光模块与 SFP+ 端口配对 |
| L2：引用完整性 | 3 | Mate 引用不存在的 Instance |
| L3：功率预算 | 4 | PDU 输出超载 |
| L3：空间冲突 | 2 | 服务器分配到重叠 U 位 |
| L4a：通道间距 | 2 | 机柜间距低于维护通道最小值 |

Table: 违规注入类别。{#tbl:violation-categories}

`piki check` 检出 15/15（100%），假阳性为零，因为它检查语义类别（Family 类型、Mate 关系）而非几何。全部 30 条规则的 L0–L4a 验证 <200ms。

与 ACC 的对比是结构性不对称的：7 项 L2 错误依赖 Family 类型系统和 Mate 关系，IFC 几何中不存在这些信息；Mate 还能区分预期兼容与真正冲突，而 ACC 必须从几何逆向推断意图。ESA 规则以 Family 类型为索引，可跨领域移植。

## 6.4 SD-HWE-Bench

违规注入验证检测准确率，但不测试 Agent 能否从自然语言生成 ADL。**SD-HWE-Bench** 填补这一空白，任务为输入自然语言需求、输出 piki YAML，评分包括 L0–L4 通过率、交付物质量和 L6 签审，将由配套论文报告。初始领域选择电信机架：空间离散、约束以代数为主、接口可枚举、`piki check` <200ms，满足 RLVR 快速奖励需求。该基准测试信息表示假说的 P1（§3.2.1）。
