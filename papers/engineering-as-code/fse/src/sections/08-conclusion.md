# 8. 结论

本文提出 Engineering as Code（EaC）范式，将工程设计迁移到以文本声明为真相源、以规则引擎为质量门、以版本控制和包管理为协作基座的模式。三项核心贡献：ADL 设计语言（Part 原子 + PDL/PML/PLL 三层正交子语言）；ESA 验证机制（将合规规则从事后 ACC 前移到设计时，在语义类别上运行规则）；信息表示假说（工程 AI 瓶颈在于缺乏可计算设计表示而非模型能力不足）。piki 原型在三个样本上验证 ADL 表达力和 ESA 检测能力（<200ms），违规注入实验 100% 检出、零假阳性。SD-HWE-Bench 设计已完成，作为假说的实证测试平台。未来工作包括提升 PLL 连续 DOF 求解精度、扩展更多工程领域、探索 EPM/AssemblyHub，以及开展 RLVR 训练实验。
