# 信息表示假说

> **信息表示假说**：传统工程领域 AI 设计生成能力滞后的根本原因，不在于模型能力不足或物理复杂性过高，而在于缺乏一种可被机器直接消费、生成、校验与版本化的设计表示——即“Design as Code”形式的可计算基础。

该假说源于对代码、数学与芯片三个领域的观察。DeepSeek-R1 在数学推理上展现出系统性链式思考能力 [@deepseek2025r1]；Claude Code 等编码智能体在 SWE-Bench 上解决了大量真实 GitHub 问题 [@anthropic2025claude]；AMS-IO-Bench 上的模型在芯片 I/O 环设计的 DRC/LVS 检查中达到高通过率 [@liu2025amsio]。这些领域的共同特征是：存在一种可被自动、快速、确定性校验的“正确性表示”——数学问题的最终答案、代码的编译与测试套件、芯片网表的设计规则 [@lightman2024letsverify]。

Lightman 等人在数学推理中发现，过程奖励模型（PRM）显著优于结果奖励模型（ORM），说明**步骤级可校验性本身是能力跃迁的关键使能条件** [@lightman2024letsverify]。SWE-RL 进一步证明，当 bug 修复的正确性可由测试套件确定性判定时，强化学习能显著提升修复成功率 [@sweagent2025swerl]。DeepSeekMath 则系统性地将这一范式命名为“可验证奖励强化学习”（RLVR），用可验证答案作为奖励信号，消除了昂贵的人工标注需求 [@shao2024deepseekmath]。

这三个案例共同构成一条因果链：

```text
结构化可验证正确性表示 → 秒级确定性反馈 → RLVR 训练有效 → Agent 能力跃迁
```

传统工程领域恰恰缺少这条链的起点。CAD/BIM 系统将设计逻辑、几何表示与工具实现紧密耦合，导致设计意图对机器不可读；Automated Compliance Checking（ACC）虽然能在下游进行反向审计，但无法弥补表示层本身的信息损失 [@eastman2009acc; @zhang2019acc]。因此，智能体在生成设计时，被迫从紧密耦合的几何表示中反向推断设计意图，无法获得代码编译或芯片 DRC 所提供的毫秒级确定性反馈，从而无法建立有效的 RLVR 训练闭环。

**可测试预测**。若信息表示假说成立，则即使底层模型保持不变，只要为智能体提供一种文本原生、机器可校验、结构解耦的“Design as Code”表示，其设计生成能力就应出现可度量的提升。本文以一个小规模试点实验对该预测进行初步验证：使用 piki 对三个样本项目执行 L0–L4a 规则检查，并统计检查延迟、规则覆盖率与生成物完整性（详见附录 III-G 与附录 V-C）。一个更系统的能力评估基准 **SD-HWE-Bench** 正在设计中：它评估智能体从自然语言工程需求生成结构化 ADL 声明的能力，并以 L0–L4 规则检查通过率作为客观奖励信号；其完整实验与大规模评估将作为后续工作（论文 B）发表。

需要强调的是，该假说并不否认物理直觉、硬约束与隐性知识的重要性。它指出的是一个更前置的问题：在讨论如何让 AI 具备工程直觉之前，工程领域首先需要一种让机器能够读取、写入和校验设计意图的符号基础。
