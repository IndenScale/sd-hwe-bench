# 回归测试与下游验证集成

Pre-commit 与 CI 专注于快速反馈，但工程设计还需要**回归测试**来捕捉跨提交、跨模块的渐进式退化。回归测试通常以 Nightly Build 或定时任务的形式运行，覆盖完整样本库、精确几何与物理仿真。它与下游验证工具的集成不是两个独立话题：L4b 之后的回归测试本质上就是通过 CAD/CAE/CFD 等外部工具对 ADL 声明进行周期性验证。

## 为什么需要回归测试

ADL 的正交性虽然减少了变更的意外副作用，但以下情况仍可能引入回归：

- 修改一个 Part Family 的字段可能影响所有引用它的 Instance；
- 新增一条 ESA 规则可能使原本通过的旧样本失败；
- 生成器模板变更可能导致下游交付物格式变化；
- 几何或物理仿真阈值调整可能暴露新的边界情况。

回归测试通过固定一组 golden samples 并在每次重大变更后重新执行，确保新功能不会破坏已有设计。

## 回归测试的层级

一个 EaC 回归测试流水线可分为三个层次：

### 1. 全样本规则回归（L0–L4a）

对所有存档样本项目运行 ADL 加载期校验（L0–L1）与 ESA 规则检查（L2–L4a），生成：

- 通过 / 失败 / 警告数量趋势；
- 新增失败的规则 ID 与样本；
- 检查延迟分布（用于发现性能退化）。

这一层不依赖外部工具，可在 CI 或 Nightly 中低成本运行。

### 2. 精确几何回归（L4b）

L4a 的快速几何（AABB、U 位冲突）只能捕捉离散或近似空间违规。对于需要精确干涉、间隙、可达性分析的场景，必须调用 CAD 内核或几何库。例如：

- 机柜内设备是否存在实际三维干涉；
- 线缆弯曲半径是否满足最小要求；
- 维护空间是否允许工具与人员进入。

L4b 检查比 L4a 慢 1–3 个数量级，不适合每次提交运行，但适合夜间回归。集成模式通常为：

```text
ADL → CAD exporter（USD/glTF/STEP）→ 干涉检查 → interference_report.json → ESA L4b
```

### 3. 物理仿真回归（L5）

对高风险的物理问题进行 CAE/CFD 仿真，例如：

- 机柜热仿真：确认功率密度下的散热是否满足要求；
- 结构仿真：确认设备重量分布下机柜变形是否在允许范围；
- 流体仿真：确认管道布局下的压降与流速是否合规。

这些仿真通常需要分钟到小时级别，产出结构化报告并与基线比较。集成模式通常为：

```text
ADL → CFD/CAE mesher（INP/MESH）→ 求解器 → thermal_report.json / stress_report.json → ESA L5
```

## 与下游工具的集成模式

下游工具与 EaC 的集成可分为三种模式：

1. **导出—执行—回写**：ADL 声明被转换为工具可读格式，工具执行分析后将结构化报告回写到项目。
2. **工具即生成器**：CAD 生成器读取 ADL 输出 3D 模型，BOM 生成器输出采购清单。生成器本身不验证，但其输出是下游流程的输入。
3. **验证即服务**：对于需要专用许可证或 HPC 资源的仿真，通过异步任务 ID 追踪状态，结果返回后更新 PR 检查状态。

## 结果的可追溯性

无论采用哪种模式，下游验证结果都应与特定 ADL 版本关联。最小可追溯信息包括：

- ADL 提交哈希；
- 工具名称与版本；
- 输入文件与参数；
- 输出指标与阈值；
- 通过 / 失败判定。

这些信息应写入版本控制或制品仓库，而不是散落在个人工作站上。

## 回归基线管理

回归测试需要明确的基线：

- **严格基线**：所有样本必须 100% 通过；
- **趋势基线**：允许已知失败存在，但不允许引入新的失败；
- **豁免基线**：对已知问题记录豁免，并在未来修复计划中追踪。

对于快速发展的 EaC 原型，趋势基线通常最实用：它允许当前边界内的已知失败存在，但要求新变更不得扩大失败集合。

## 与豁免机制的联动

下游验证结果常常触发豁免。例如，L4a 快速几何报告某处间隙不足，但 L4b 精确几何仿真证明该间隙在装配公差内可接受。此时，工程师可以基于 L4b 报告对 L4a 规则申请豁免，并将仿真报告作为豁免记录的附件。这种联动使 ESA 的“规则可豁免”纲领具备工程可操作性。

## 工具中立性

EaC 对下游工具保持中立。项目可以使用开源几何内核（如 OpenCASCADE）、商业 CAD（如 SolidWorks、Revit）、专业 CAE（如 ANSYS、Abaqus）或自研求解器，只要它们能消费 ADL 导出的结构化输入并返回结构化输出。工具是消费者，ADL 是真相源。

## 与 EaC 校验层级的对应

| EaC 层级 | 范围 | 验证形式 | 典型频率 |
|---|---|---|---|
| L0–L1 | ADL 加载期校验 | Pre-commit / CI | 每次提交 |
| L2–L3 | ESA 核心 | Pre-commit / CI | 每次提交 / 每次 PR |
| L4a | ESA 快速几何扩展 | CI / Nightly | 每次 PR / 每晚 |
| L4b | 精确几何（CAD 内核） | Nightly / Weekly | 每晚或每周 |
| L5 | 物理仿真（CFD/FEA） | Weekly / On-demand | 每周或按需 |
| L6 | AI / 人工审批 | Release gate | 发布前 |

这一分层确保昂贵验证只在必要时运行，而廉价验证持续提供快速反馈。
