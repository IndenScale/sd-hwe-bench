# ESA 的四条操作性纲领

Engineering Static Analysis（ESA）的核心目标是在设计声明提交或持久化之前，通过确定性规则引擎完成**可快速执行、不依赖物理仿真**的规则校验，从而将质量保证活动从设计流程的右端前移到早期阶段。ESA 主要覆盖 L2（引用完整性、Mate 自洽性、接口兼容性）与 L3（跨记录业务规则）；L4a（快速几何，如 AABB、U 位冲突）可作为其轻量扩展。L0–L1 属于 ADL 加载期解析与 schema 校验，L4b–L6 属于精确几何、物理仿真与人工审批等下游验证，均不属于 ESA 范畴。

这一思路与芯片设计中的 Design Rule Checking（DRC）一致：规则检查嵌入布局与布线工具，非合规几何构型在生成时即被拦截，而非等待事后审查 [@ieee1364]。DRC 本身也不执行电路仿真或版图人工签收，而是专注于快速、可判定的几何规则。

为适配真实工程实践的复杂性，ESA 遵循四条操作性纲领：

## 纲领 I：规则可豁免

物理世界的边界条件无法被有限规则集穷尽。ESA 允许具有相应授权的工程人员豁免特定规则，但每一次豁免都必须作为设计历史的一部分被显式记录，并触发下游强化校验（例如更高精度的 CAE 仿真或物理测试）。该机制将“超规范”行为从非正式决策转化为可追溯、可审计的风险管理流程。

**操作示例**。假设规范要求机柜维护通道宽度不小于 600 mm，但因场地限制某项目必须采用 500 mm。工程师可在 `waiver.yaml` 中记录：对象 `RACK-A01/RACK-A02`、规则 `TELECOM-FLOOR-002`、理由“受既有建筑柱网限制”、授权人、以及下游任务“提交 CFD 热仿真确认无散热风险”。豁免记录随 Git 提交进入版本历史，PR 审查者可逐条审阅。

## 纲领 II：聚焦底线规则

ESA 初期部署应聚焦于构成大量重复性检查基础的机械性、确定性条款：防火间距、疏散净宽、最小净高、功率预算、U 位冲突等。规则库应支持模块化配置，允许项目在国家规范、地方标准、企业内部控制等不同层级组合规则包。底线规则的优势在于二值化判断明确、争议空间小、自动化收益高。

## 纲领 III：左移与信噪比优化

软件工程中的“左移”质量理念已被 CI/CD 实践广泛验证：lint 与类型检查在提交前拦截低价值错误，使代码审查能够聚焦于架构与逻辑 [@chiari2024iacstatic]。ESA 将同一理念迁移到工程设计：在提交前拦截功率预算越界、U 位冲突、接口不匹配等低级错误，使下游人类专家和高昂分析工具能够专注于真正需要专业判断的问题，而非被低价值错误分诊消耗。

## 纲领 IV：AI 辅助规则库构建

规则库构建是 EaC 落地的关键瓶颈。在专家监督下，大语言模型可从规范文本、历史审查意见和设计指南中提取初步形式化规则草案 [@eastman2009acc; @zhang2019acc; @yang2024llmacc; @nakhaee2024kgllm]，降低规则工程的初始成本。长期看，规则库应采取开源社区治理模式：核心规则由权威机构维护，领域规则由从业者贡献，通过版本控制和认证机制保证质量。

四条纲领相互支撑：可豁免性让规则集在不完美世界中可用；聚焦底线让初期落地可行；左移优化让反馈价值最大化；AI 辅助让规则库可持续扩展。
