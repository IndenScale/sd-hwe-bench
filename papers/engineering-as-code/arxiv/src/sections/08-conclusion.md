# 8. 结论

本文提出了 Engineering as Code（EaC）范式：将工程设计从以 GUI 为中心的 WYSIWYG 模式，迁移到以文本声明为真相源、以规则引擎为质量门、以版本控制和包管理为协作基座的模式。

本文的三项核心贡献如下：

1. **ADL 设计语言**。以 Part 为原子、以 PDL/PML/PLL 三层正交子语言为核心的装配体定义语言。ADL 明确各层语法与语义边界，使工程设计意图以文本原生的声明式形式表达，并直接与 Git、CI 和 LSP 风格诊断集成。PLL 通过参数化自由度补全和骨骼建模机制，使配合约束求解与装配体层级管理在同一框架中正交共存。
2. **ESA 验证机制**。将合规性规则从下游 ACC 的事后审查前移到设计生成阶段。与模型 ACC 不同——后者受困于几何碰撞假阳性和命名依赖翻模等结构性缺陷——ESA 检查的是设计声明而非几何实例化产物，因此能在语义类别上运行规则而不依赖项目命名约定。给出了规则可豁免、聚焦底线规则、净化协作信噪比、AI 辅助规则库建设四条操作性纲领。定义了 L4b–L6 下游验证的 CI/CD Action 集成协议——该协议是 EaC 工作流架构的完整性声明，确保 ESA 的确定性边界与物理工程所需的完整验证生态互补而非冲突。
3. **信息表示假说**。系统论证工程 AI 瓶颈的根源不在于模型能力不足，而在于传统工程领域长期缺乏"Design as Code"形式的可计算基础。ACC 的假阳性和翻模开销是这一论断的具体证据：当设计意图被困在几何模型中时，不仅 AI 无法获得训练信号，连人类工程师的自动化检查都在语义层面失效。

piki 原型在三个样本上验证了 ADL 的表达力和 ESA 的检测能力（共 69 项规则通过，<200ms 延迟）：电信机架扩容（0 错误 1 警告，29 通过）、模块化集装箱数据中心（0 错误，13 通过）、机械键盘装配（0 错误，27 通过）。违规注入实验以 100% 检出率和零假阳性完成。SD-HWE-Bench 评测基准的设计已完成，作为信息表示假说的实证测试平台。

未来工作包括：提升 PLL 连续 DOF 的几何求解精度并扩展到管线/缆索等 1D 连续路径拓扑；探索工程包管理与装配体注册中心（EPM/AssemblyHub）的设计空间；在更多工程领域验证 ADL 与 ESA 的表达能力；以及开展 RLVR 训练实验以直接测试信息表示假说。

本文的核心信息是：物理工程需要源代码。不是更好的 CAD，不是更智能的审查工具，而是一种与硬件描述语言为芯片设计所做贡献相同的东西——一种将设计意图从几何实现中解放出来、使其可被软件工程基础设施操作的表示。这正是 EaC 提供的。
