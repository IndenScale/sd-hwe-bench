# EaC 分层校验体系：ESA 与下游验证的边界

EaC 将工程设计的校验划分为七个递进层级，覆盖从文件格式到人工审批的完整质量光谱。**Engineering Static Analysis（ESA）特指其中的 L2–L3 层级，L4a 可作为其轻量几何扩展；L0–L1 属于 ADL 加载期校验，L4b–L6 属于下游验证与审批。**

## 七层校验体系

| 层级 | 范围 | 执行时机 | 是否属于 ESA | 延迟目标 |
|---|---|---|---|---|
| L0 | YAML/JSON 解析有效性 | 加载 | 否 | < 1 ms |
| L1 | Schema 校验（pydantic） | 加载 | 否 | < 10 ms |
| L2 | 引用完整性、Mate 自洽性、接口兼容性 | 加载 / `check` | **是（核心）** | < 10 ms |
| L3 | 跨记录业务规则 | `check` 默认 | **是（核心）** | 10–100 ms |
| L4a | 快速几何（AABB、U 位冲突、楼层碰撞） | `check` 默认 | 部分扩展 | 100 ms–1 s |
| L4b | 精确几何（CAD 内核干涉、间隙、可达性） | `verify geom` | 否 | 1 s–1 min |
| L5 | 物理仿真（CFD/FEA） | `verify phys` | 否 | 1 min–1 h |
| L6 | AI / 人工审批 | `signoff` | 否 | 秒级–分钟级 |

这一划分的关键在于**计算成本与确定性**：ESA 层（L2–L3，以及部分 L4a）具备毫秒级、确定性、可重复执行的特点，适合嵌入 CI/CD 与 pre-commit；L4b 之后需要调用外部 CAD 内核或数值求解器，成本高、参数敏感，属于下游验证而非静态分析。

## 为什么 ESA 主要是 L2–L3

L2 与 L3 是 ESA 的核心，因为它们满足静态分析的三个特征：

1. **不执行物理仿真**：只基于声明之间的关系和约束进行推理；
2. **结果确定性**：给定相同的 ADL 声明，总是产生相同的通过 / 失败判定；
3. **成本低廉**：可以在毫秒到百毫秒级完成，适合高频率执行。

L4a（快速几何）是边界案例。它虽然涉及空间计算，但使用的是 AABB、离散 U 位、代数不等式等轻量方法，不调用 CAD 内核，因此可以视为 ESA 向几何领域的有限扩展。一旦需要精确干涉、间隙分析或连续 3D 约束求解（L4b），则超出静态分析范畴，进入几何验证领域。

## 与软件工程验证体系的映射

EaC 的分层校验与软件工程验证体系形成精确映射，其中 ESA 对应于类型检查之后的静态分析与轻量测试：

| 软件工程 | EaC 层级 | 检查内容 | 是否属于 ESA |
|---|---|---|---|
| Parser | L0 | 文件格式有效性 | 否 |
| Type checker | L1 | Schema 校验、值域检查 | 否 |
| Link checker / Linter | **L2** | 引用完整性、Mate 自洽性 | **是** |
| Unit test + integration test | **L3** | 跨记录业务规则 | **是** |
| End-to-end test / smoke test | **L4a** | 快速几何碰撞、空间边界 | **部分** |
| Performance benchmark / simulation | L4b–L5 | 精确几何、CFD/FEA | 否 |
| Code review / sign-off | L6 | 交付物校验、综合评估 | 否 |

这一映射说明：ESA 并非要覆盖所有验证活动，而是专注于**可在设计阶段快速、确定性执行的规则检查**。更昂贵的验证留给下游专用工具，这正是 EaC 与“万能工具”式 CAD/CAE 集成的区别。

## 当前实现边界

piki 原型目前完整覆盖 L0–L4a：

- L0–L1 在 ADL 加载阶段运行，属于解析与 schema 校验；
- **L2–L3 作为 ESA 核心**，由规则引擎执行；
- **L4a 作为 ESA 的几何扩展**，使用快速几何方法；
- L4b（精确 CAD 干涉）、L5（物理仿真）与 L6（AI/人工审批）尚未实现，属于下游验证与未来工作。

L2–L4a 已足以支持电信机架等离散场景并验证“质量左移”假设。论文正文将诚实陈述：ESA 的核心是 L2–L3，L4a 为可选扩展，L4b–L6 不在当前 ESA 范围内。
