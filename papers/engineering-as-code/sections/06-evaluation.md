# 6. 评估

本章报告当前评估状态和实验计划。**标记为"进行中"的实验结果正在整理，将在投稿截止日前完成补充。**

## 6.1 研究问题与方法论

我们回答三项研究问题：

- **RQ1.** ADL 能否以文本原生、可被机器检查的形式表达真实工程设计？
- **RQ2.** ESA 能否在设计时检测常见设计违规，延迟满足 CI/CD 门禁要求，且信噪比高于基于几何的 ACC？
- **RQ3.** 该表示是否提供了 RLVR 式 Agent 训练所需的可验证奖励信号？

评估使用 piki 仓库中的样本项目和一项受控的违规注入实验。测量指标包括：

1. **表达力**：每个样本能否完全以 ADL 声明；
2. **检查延迟**：`piki check` 完成 L0–L4a 验证的时间；
3. **规则覆盖**：适用的规则数量和类型；
4. **检测准确率**：ESA 是否在语义合理的配置中识别出故意注入的违规而不产生假阳性；
5. **交付物生成**：下游工件（BOM、面板视图、端口映射、线缆清单）能否产出。

所有测量在笔记本级机器上以热缓存进行；延迟为中位数五次运行。

## 6.3 违规注入实验 [进行中]

为测试 ESA 的检测准确率并对比基于模型的 ACC 基线，我们正在电信样本上进行受控违规注入实验。

**实验设计**。在正确的 ADL 声明中注入 15 个已知违规，覆盖四个类别：

| 类别 | 数量 | 示例 |
|------|-----|------|
| L2：接口不匹配 | 4 | SFP28 光模块与 SFP+ 端口配对 |
| L2：引用完整性 | 3 | Mate 引用不存在的 Instance |
| L3：功率预算 | 4 | PDU 输出超载超过额定容量 |
| L3：空间冲突 | 2 | 两台服务器分配到重叠 U 位 |
| L4a：通道间距 | 2 | 机柜间距低于维护通道最小值 |

然后分别运行 `piki check`（ESA）和基于模型的 ACC 基线——从同一 ADL 声明通过 piki CAD 导出器导出为 IFC，再用代表性 ACC 工具检查。

**预期结果**：

1. **ESA 检测率**：15/15 违规全部检出（100%），假阳性为零——因为 ESA 检查语义类别（Family 类型、Mate 关系）而非几何。
2. **ACC 基线**：将检测空间冲突（L3 空间、L4a 通道），但对接口不匹配和引用完整性违规将遗漏或产生模糊结果，因为这些需要语义信息而几何模型不保留。同时将在语义合理的几何交点中产生假阳性（如线缆穿过机柜理线孔）。
3. **检查延迟**：ESA 在 200ms 内完成所有检查；ACC 管道需要模型导出 + 加载 + 规则执行，在秒到分钟级。

实验直接测试 §2.2.1 和 §5.1 中的主张：ESA 的优势不仅是速度，更在于能获取基于几何的检查所缺失的语义信息。

**状态**。ADL 违规注入语料已构建。ACC 基线管道正在搭建中。完整结果正在整理，将在投稿截止日前包含。

## 6.4 SD-HWE-Bench：Agent 评测基准 [设计阶段]

试点和违规注入实验验证可行性和检测准确率，但不测试 Agent 能否从自然语言需求**生成** ADL。我们正在设计 **SD-HWE-Bench** 以填补这一空白。它将由配套论文报告；此处描述其设计以供审稿人评估更大的评估计划。

**任务范式**：

```text
输入：  自然语言工程需求
输出：  结构化 ADL 声明（piki YAML）
评分：  L0–L4 规则检查通过率 + 交付物质量 + L6 签审评估
```

与 SWE-bench 不同——后者测试对已有代码库的补丁生成 [@jimenez2024swebench]——SD-HWE-Bench 测试从零创建，因为声明式工程设计作为一种实践尚未规模化存在。

**初始领域：电信机架部署**。选择该领域是因为：

- 空间离散（机柜 U 位），避免连续 3D 约束求解；
- 约束以代数为主（功率预算、重量、U 位冲突）；
- 接口类型可枚举（SFP28、RJ45、IEC-C13/C14）；
- `piki check` 约 200ms 完成，满足 RLVR 对快速奖励信号的需求。

**计划指标**：

| 指标 | 定义 |
|------|-----|
| `Pass@1` | 首次 ADL 输出通过全部启用规则的百分比 |
| `Pass@k` | 至少一次通过 k 次采样的百分比 |
| `Latency` | 每次输出的 `piki check` 中位耗时 |
| `Coverage` | 任务需求在生成 ADL 中的覆盖率 |
| `Human sign-off` | 领域专家评定可接受的百分比（L6 代理） |

**与 RQ3 的连接**。SD-HWE-Bench 直接测试 RLVR 因果链：如果 ADL + ESA 提供了结构化、确定性的奖励信号，则使用该信号训练的 Agent 应优于无该信号的 Agent。这是信息表示假说的预测 P1（§3.2.2）。Benchmark 设计已完成；任务编写和基线 Agent 实验正在进行中。
