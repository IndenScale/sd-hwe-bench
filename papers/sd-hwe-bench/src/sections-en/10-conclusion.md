# 10. 结论

本文介绍了 SD-HWE-Bench——首个面向软件定义硬件工程的 AI Agent benchmark。SD-HWE-Bench 贡献了：(1) 一个包含 28 个任务、3 个 canonical 工程、3 个子领域的数据集，涵盖常规设计、复合依赖、涌现约束和跨专业综合四类任务；(2) 一套 L0-L6 七层 DTS 评分协议（含 v2 的 L2a/L2b/L2c 三层拆分）；(3) 一组 CLI Agent 在 28-task 全量 set 上的 pass@1 baseline。

主要发现：

- **模型间区分度已建立**：在同一 CLI Agent 框架下，Kimi k2.7（93%）与 Gemini 2.5 Flash（81%）存在 12pp 的 pass@1 差距。hard 任务（跨专业综合、涌现约束）上的差距更大（待验证 vs 待验证），说明 benchmark 已在最难的 15% 任务上产生了明确的模型排序信号。
- **规范驱动有效**：Agent 必须从设计规范文档和设备参数表中主动挖掘字段名、约束公式和目录结构的设计原则，在不同模型间产生了系统性区分——Kimi 通过频繁查阅规范文档保持高精度，Gemini 在跳过查阅步骤时产生显著更高的 L0 语法错误率。
- **L0 是意外的区分源**：Gemini 2.5 Flash 在 YAML 生成精度上的劣势（L0 通过率 85% vs Kimi 94%）是其最大单一失败原因，说明在结构化文本产出场景下，基础代码生成精度仍是真实瓶颈。
- **L3 多约束协调是核心区分维度**：跨专业任务中电气+结构+消防联合约束的协调能力是当前 benchmark 区分度最强的单一维度。
- **medium 任务群待加密**：46% 的任务（21 个 medium）对 CLI Actor 不产生区分度，是下一步约束加密和任务升级的优先方向。

SD-HWE-Bench 的代码、数据集、评分器和 baseline 结果全部开源，为 Engineering as Code 评测提供标准化基础设施。我们邀请社区在新的模型、新的 Actor 框架和新的工程领域上扩展此基准。
