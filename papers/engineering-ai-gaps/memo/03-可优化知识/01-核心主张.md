# 可优化知识：核心主张

本 memo 记录“可优化知识鸿沟”章节的核心论证。它不是正文定稿，而是后续改写 `src/sections-zh/06-knowledge-experiment.md` 与工程 AI 知识基础设施论述的依据。

## 第一性问题

当前“知识鸿沟”论述的一个潜在问题是：知识这个词太宽。工程知识可以指规范条文、设备参数、施工经验、供应商报价、气候数据、电价曲线、材料工艺窗口，也可以指模型训练语料中的一般常识。如果只说“工程 AI 缺少知识”，读者很难判断它与约束、上下文、检索增强或领域微调的区别。

因此，本章的核心问题不是“模型是否知道更多工程事实”，而是：

> **什么样的工程知识能够改变设计上限？**

我们的回答是：**可优化知识**（optimizable knowledge）。可优化知识不是静态背景资料，而是能够进入设计变量、模型参数、目标函数、约束接口或搜索空间，并改变 Pareto frontier 的工程知识。

## 与可执行约束的关系

可执行约束与可优化知识不是并列替代关系，而是递进关系。

- **可执行约束定义可行域**：回答方案是否违反工程底线，例如接口是否兼容、容量是否超限、间距是否满足、施工资源是否冲突。
- **可优化知识塑造目标景观**：回答在可行域内哪些方案更接近前沿，例如不同设备曲线、气候、电价、水价、报价、交期、故障率和调度策略如何共同影响 TCO、PUE、SLA、工期和风险。

一句话：

> 可执行约束证明方案没有错；可优化知识解释方案为什么更好。

或者更精确地说：

> 可执行约束刻画工程可行域，可优化知识塑造目标景观。前者使 Agent 避免 pseudo-correctness，后者决定 Agent 是否能够从合规方案进入前沿方案。

## 从“背景知识”到“可优化知识”

许多工程知识虽然存在，但没有进入优化闭环：

- 设备 datasheet 中有 COP 曲线、效率曲线和运行边界，但 Agent 只看到型号名称。
- 供应商报价中有 CAPEX、交期、维保成本和折扣条件，但设计过程只使用静态容量规则。
- 气候数据有全年湿球温度和极端天气分布，但冷却系统只按峰值工况校核。
- 电价和水价有时序结构，但运营策略只按平均价格估算成本。
- 施工经验包含吊装窗口、天气风险、资源瓶颈和应急预案，但排程只检查活动依赖是否完整。

这些信息只有在被转化为可计算对象后，才成为可优化知识。可优化知识至少需要满足：

- **可参数化**：能作为模型参数、曲线、场景或候选库进入计算。
- **可组合**：能与设备选型、布局、调度、施工和成本模型联动。
- **可权衡**：能进入目标函数或多目标 Pareto 分析，而不是只做 pass/fail。
- **可更新**：能随供应商、地点、价格、气候、工艺和项目经验变化而版本化更新。
- **可追溯**：优化结果能追溯到使用了哪些知识源、曲线、报价或场景假设。

## 与 PDK 的类比

半导体 PDK 的价值不只是“提供知识”，而是把工艺能力转化为 EDA 可消费的规则、模型、参数和库，使设计工具能够在真实工艺边界内搜索。

系统级工程缺少的不是更多 PDF，而是 PDK-like knowledge infrastructure：

- 设备厂商知识需要变成候选设备库、效率曲线、价格曲线和可用性约束。
- 地点知识需要变成气候时序、电价时序、水价时序和政策场景。
- 施工知识需要变成资源容量、天气窗口、吊装约束、应急策略和风险模型。
- 运营知识需要变成负载曲线、调度策略、SLA 风险和生命周期成本模型。

当这些知识进入可闭环表征和可执行评估后，Agent 才能进行跨设备、跨布局、跨调度、跨施工和跨成本的协同优化。

## 章节要证明的核心结果

本章应围绕四个主结果组织证据：

1. **静态合规不等于前沿设计。**

   通过同一 AIDC 任务比较静态容量/冗余/阈值检查与动态时序经济建模，证明满足规则的方案仍可能在 TCO、PUE、SLA 或风险上显著落后。

2. **可优化知识会改变设计选择。**

   当引入气候曲线、电价曲线、水价曲线、负载曲线、设备效率曲线和报价后，最优设备组合、储能容量、冷却策略和运营调度可能发生变化。若设计选择发生可解释变化，说明知识不只是上下文，而是进入了决策。

3. **可优化知识会推动 Pareto frontier 外移。**

   如果 Dynamic Knowledge 条件在相同 SLA 下获得更低 TCO，或在相同 CAPEX 下获得更低 OPEX/PUE/风险，说明可优化知识改变了解空间上限。

4. **知识基础设施决定 Agent 能否从合规走向最优。**

   即使 Agent 能通过可执行约束，如果缺少可优化知识，它仍只能停留在保守合规区间；只有知识被模型化、版本化并接入优化目标，Agent 才能发现高性能区域。

## 章节主张的一句话版本

> 工程 AI 缺少的不是一般意义上的领域知识，而是可优化知识：能够进入设计变量、动态模型、目标函数和搜索空间，并在可执行约束定义的可行域内推动 Pareto frontier 外移的工程知识。
