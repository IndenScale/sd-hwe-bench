# 2. 从三重鸿沟到 Evaluation Substrate

本章将三重鸿沟重新定义为 Engineering as Code evaluation substrate 的三个设计要求。它们不是并列的问题清单，而是递进关系：表征决定 agent 能提交什么，约束决定系统能否判定真实正确，知识决定正确解空间是否接近前沿实践。

## 2.1 表征鸿沟：从可计算到可闭环

表征鸿沟不是简单地说工程缺少可计算表示。参数化设计、CAD family、BIM 族、OpenSCAD、Grasshopper、Dynamo 和 Excel 配置表都已经证明，工程工件可以被变量驱动、批量生成、版本化和复用。真正的问题是：**什么样的可计算表征才配称为 engineering agents 的 evaluation substrate？**

本文的回答是：可闭环表征。可计算只是必要条件，可闭环才是充分条件。一个可闭环工程表征至少需要支持五类闭环：

- **生成-验证闭环**：表征不仅能生成工件，还能被映射到可执行 critic。
- **本专业-跨专业闭环**：电气、热、结构、布局、施工和运营不是各自正确，而是在同一语义层被联合检查。
- **设计-建造闭环**：设计状态能接受吊装、施工通道、资源窗口、EPC 排程等建造反馈。
- **当前项目-未来项目闭环**：工程知识可以跨项目迁移，而不是绑定某个作者的脚本习惯。
- **个人知识-组织知识闭环**：隐式约定被显式化为对象、关系、接口、约束和依赖。

因此，EaC 不把自己定义为 CAD/BIM/参数化设计的替代品，而是定义为 agent 提交与评估的语义层。CAD 可以表达几何，BIM 可以组织工程文档，OpenSCAD 可以表达 code-like geometry，MCP 可以暴露工具接口，CUA 可以操作 GUI；但 evaluation substrate 还必须规定：工程状态如何被提交，引用关系如何被检查，错误如何定位，repair 如何发生。

## 2.2 约束鸿沟：从规范文本到可执行正确性

约束鸿沟指工程正确性大量存在于自然语言规范、人工经验和隐式审查流程中，而没有变成可执行、可定位、可归因的 critic。自然语言规范可以告诉人“应当满足什么”，但不能稳定回答三个评估问题：本次任务涉及哪些约束，提交物是否真实满足这些约束，错误发生在哪个对象、字段、关系或约束层。

这导致 engineering agents 的核心失败形态不是明显胡说，而是 pseudo-correctness：方案在语言上完整、专业、结构合理，却在可执行检查下失败。例如：

- 引用不存在的设备、接口或端口。
- 连接器、线缆、光模块或接口类型不兼容。
- 功率预算、结构载荷、接地、防雷、消防或维护间距被遗漏。
- 布局满足视觉直觉，却违反吊装半径、施工路径或 VDC 工作面前置条件。
- 调度计划看似完整，却违反资源容量、天气窗口、交期或 SLA。

EaC 的约束层将正确性从“模型是否记得规则”转化为“系统是否能判定并反馈规则”。这并不意味着模型不重要；它意味着评估对象从单次自然语言输出变成 agent 与 critic 共同构成的闭环系统。同样的错误，如果只反馈“设计不满足电气要求”，repair 很可能退化为猜测；如果反馈到文件、对象、字段、期望值、实际值和约束族，agent 才能进行局部修复。

## 2.3 知识鸿沟：从背景资料到可优化知识

知识鸿沟比前两者更深。即使工程状态可提交、约束可执行，agent 仍可能被保守、静态或滞后的知识边界限制。这里的“知识”不是泛指更多上下文、更多检索文档或更多领域语料，而是指能进入设计变量、模型参数、目标函数、约束接口或搜索空间，并改变 Pareto frontier 的 **可优化知识**。

可执行约束定义可行域：接口是否兼容、容量是否超限、间距是否满足、资源是否冲突。可优化知识塑造目标景观：不同设备曲线、气候、电价、水价、报价、交期、故障率和调度策略如何共同影响 TCO、PUE、SLA、工期和风险。简言之，可执行约束证明方案没有错；可优化知识解释方案为什么更好。

半导体 PDK 是一个重要对照。PDK 的价值不只是提供知识，而是把工艺能力转化为 EDA 可消费的规则、模型、参数和库，使设计工具能够在真实工艺边界内搜索。许多系统级工程领域缺少 PDK-like knowledge infrastructure：设备厂商知识没有变成候选设备库和效率曲线，地点知识没有变成气候与电价时序，施工经验没有变成资源容量、天气窗口和风险模型。

AI Data Center Design 适合作为知识鸿沟的 reference domain。设备选型、冷却布局、储能容量、光伏配置、当地气候、电价结构、负载曲线和调度策略之间存在强耦合。局部满足规则不等于全局最优；只有当这些知识进入 EaC substrate 的模型、目标和搜索空间，agent 的评估才可能从合规走向前沿。
