# 6. 实验三：可优化知识

知识实验关注的不是能否通过规则检查，而是知识是否改变设计选择和优化上限。我们以 AI Data Center Design 作为 probe，测试当设备曲线、气候、电价、水价、负载、成本和施工风险被转化为模型参数、目标函数和搜索空间后，agent 是否能从合规方案走向前沿方案。

## 6.1 第一性问题

“知识”这个词太宽。工程知识可以指规范条文、设备参数、施工经验、供应商报价、气候数据、电价曲线、材料工艺窗口，也可以指模型训练语料中的一般常识。本文关注的不是“给 agent 更多资料是否更好”，而是：

**什么样的工程知识能够改变设计上限？**

我们的回答是：可优化知识。可优化知识不是静态背景资料，而是能够进入设计变量、动态模型、目标函数、约束接口或搜索空间，并改变 Pareto frontier 的工程知识。设备 datasheet 中的 COP 曲线、报价中的 CAPEX 与维保成本、全年湿球温度、分时电价、施工天气窗口、吊装资源瓶颈，只有被转化为可计算对象后，才成为 evaluation substrate 的一部分。

## 6.2 为什么选择 AI Data Center Design

AIDC 同时包含设备选型、冷却系统、供配电、储能、光伏、机柜布局、当地气候、分时电价、施工约束和运营调度。局部满足规则并不意味着全局最优。例如，更高 CAPEX 的冷却设备或储能系统可能允许更激进的调度策略，降低全年 OPEX 或 SLA 风险；当地湿球温度会改变 free cooling 潜力；电价结构会改变储能容量与充放电策略；设备供应商曲线会改变 PUE/TCO 最优点。

这些知识很难固化为简单 rule check。规则可以判断是否违反最低要求；知识实验关心的是，在相同可行域内，情境化知识是否改变目标景观，并推动 Pareto frontier 外移。

## 6.3 知识条件

在同一 AIDC 设计-运营任务上设置多组知识条件。所有条件都使用同一套硬性可执行约束作为最终可行性闸门，差异只在可优化知识是否进入目标函数、模型参数和搜索空间：

| 条件 | Agent 可用信息 | 评估方式 | 目的 |
|---|---|---|---|
| Static Compliance | 静态容量、冗余、间距、PUE/SLA 阈值 | pass/fail + 静态评分 | 测试只靠合规能达到的上限 |
| Static + Cost | 增加设备报价、CAPEX/OPEX 参数 | LCC/TCO 评分 | 测试成本知识是否改变选型 |
| Static + Curves | 增加设备效率曲线、冷却曲线、负载曲线 | 准动态仿真 | 测试工程曲线是否改变设备组合 |
| Dynamic Knowledge | 增加全年气候、电价、水价、负载、设备报价和效率曲线 | 时序仿真 + TCO/PUE/SLA | 测试完整可优化知识是否推动 frontier |
| Oracle Optimizer | 使用同一知识库的参考优化器 | 搜索得到参考 Pareto frontier | 给 agent 方案提供上限参照 |

关键控制原则是：最终可行性约束不变。知识条件改变的是可优化信息和评分目标，而不是降低或提高工程底线。所有条件最终都用完整动态评估器离线重评，避免静态条件因为评分较弱而虚高。

## 6.4 优化权限

AIDC probe 还需要按可优化变量分层，观察知识进入后是否真的触发设计-运营联合优化：

| 设置 | 可优化变量 | 观察对象 |
|---|---|---|
| Fixed Design | 设备和布局固定，只优化运营调度 | 调度策略对 OPEX/SLA 的影响 |
| Equipment-only | 优化冷机、储能、变压器、光伏、机柜/服务器候选 | CAPEX/OPEX 权衡 |
| Schedule-only | 设备固定，优化温度设定点、储能充放电、负载迁移 | 峰谷套利、free cooling、SLA 风险 |
| Joint DTCO | 设备选型 + 布局 + 运营调度联合优化 | Pareto frontier 是否显著外移 |

如果 Joint DTCO 明显支配单独优化条件，说明可优化知识的价值来自跨层组合，而不是某一条规则或某一个参数。

## 6.5 指标与判断标准

本实验不以 pass rate 为主指标。pass/fail 只用于确保方案处于可行域内，真正的观测对象是 frontier 和目标权衡：

- **Feasibility rate**：满足硬性可执行约束的比例。
- **TCO**：生命周期总成本。
- **CAPEX / OPEX**：投资与运营成本拆分。
- **PUE / annual energy**：全年能耗和冷却效率。
- **SLA violation rate**：温控、供电、容量或服务水平违规率。
- **Water usage / water cost**：用水量与水费。
- **Risk metrics**：极端天气、价格波动、供应延迟或设备故障下的风险。
- **Pareto dominance**：不同知识条件下方案之间的支配关系。
- **Frontier shift**：Dynamic Knowledge 相对 Static Compliance 的 Pareto frontier 外移程度。
- **Decision shift**：设备组合、储能容量、冷却策略、调度策略是否发生可解释变化。

可优化知识的重要性可以通过三类证据证明：同可行性下性能提升，设计决策发生可解释变化，以及 Dynamic Knowledge 或 Joint DTCO 条件产生的非支配解集相对静态条件外移。

## 6.6 与 PDK 的类比

半导体行业之所以能进行快速 DTCO，是因为 PDK 把工艺能力、规则和模型传递给设计工具。许多系统级工程没有类似机制：设备厂商、材料供应商、施工单位、业主和运营方的知识被分散保存，难以被 agent 或优化器统一消费。

AIDC probe 的意义在于展示：如果缺少 PDK-like knowledge infrastructure，agent 只能在保守规则下做局部合规；如果情境化知识能进入 EaC substrate 的设计变量、动态模型和优化目标，agent 才可能进行跨设备、跨布局、跨运营策略和跨施工风险的协同设计。
