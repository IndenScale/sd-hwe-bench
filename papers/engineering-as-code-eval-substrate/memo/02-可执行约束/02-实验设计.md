# 可执行约束：实验设计

本 memo 记录“可执行约束鸿沟”章节的实验设计。实验目标不是只证明 Executable 条件优于 NL-only，而是拆解两个正交变量：约束可执行覆盖率与诊断信息粒度。

## 实验目标

实验需要支撑以下问题：

- 自然语言任务描述和规范文档是否足以让 Agent 稳定满足工程约束？
- 当约束被转化为可执行 critic 后，Agent 是否更可靠地达到真实正确？
- 可执行覆盖率从低到高变化时，设计质量、遗漏密度和 repair 收敛是否随之变化？
- 诊断是否包含定位和归因信息，对修复成功率和效率有多大影响？
- 这些结论在不同任务规模和约束耦合复杂度下是否稳健？

## 轴 A：约束可执行覆盖率

约束可执行程度应作为连续变量，而不是只有三种固定条件。

| 条件 | Agent 可见内容 | 可执行检查 | 目的 |
|---|---|---|---|
| NL-only | 任务描述中的自然语言约束 | 无 | 测试自然语言要求是否足够 |
| Docs-only | 任务描述 + 规范文档 | 无 | 测试大型规范是否会被正确提取和落实 |
| Partial executable | 部分约束进入提示词或 repair 反馈 | 完整评分，部分反馈 | 测试可执行反馈覆盖率与遗漏率的关系 |
| Full executable | 所有适用约束进入提示词或 repair 反馈 | 完整评分，完整反馈 | 测试完整可执行闭环能否稳定收敛 |

Partial executable 条件可以通过以下方式构造。注意：最终评分始终使用完整约束集合，partial 只控制约束对 Agent 的可见性或反馈可见性。

- 随机对 Agent 隐藏一部分约束，记录隐藏约束是否被模型遗漏。
- 按约束族控制可见性或反馈，例如电气、热、结构、布局、施工、调度、交付物。
- 按评分层控制反馈，例如只反馈 L0-L2、L0-L3、L0-L4、L0-L5 的诊断，但最终仍用 L0-L5 完整评分。
- 保持 critic 能发现并计分所有错误，但不把部分 diagnostic 反馈给 Agent，用于区分“评估覆盖”和“修复反馈覆盖”。

核心指标：

- pass@1。
- repair 后 pass rate。
- pseudo-correctness rate。
- omission density。
- 对 Agent 不可见或未反馈约束上的真实违规率。
- constraint coverage 与 final quality 的关系曲线。
- repair saturation curve。

## 轴 B：诊断信息粒度

在相同 critic 和相同错误集合下，控制反馈给 Agent 的 diagnostic 粒度。

| 条件 | 反馈内容 | 目的 |
|---|---|---|
| No diagnostic | 只告诉 failed | 测试无定位反馈下 repair 是否退化为猜测 |
| Coarse diagnostic | 自然语言摘要，不含精确对象/字段 | 测试普通审查意见的效果 |
| Attributed diagnostic | 给出约束族、对象 id、失败原因 | 测试归因信息的作用 |
| Localized diagnostic | 给出文件、字段、对象、期望值、实际值、修复建议 | 测试可定位诊断的上限 |

核心指标：

- repair success rate。
- repair rounds used。
- time/token cost。
- 二次破坏率。
- 同类错误重复率。
- 从首次发现到最终修复的收敛轨迹。

## 任务分层

实验任务不能只按大小采样，需要按“规模”和“耦合复杂度”分层。

| 任务类型 | 规模 | 耦合复杂度 | 预期现象 |
|---|---:|---:|---|
| 简单布置/连接 | 低-中 | 低 | NL 也可能高通过；executable 主要提升稳定性 |
| 大量对象但弱耦合 | 高 | 低-中 | executable 降低偶发遗漏和回归 |
| 跨文件/跨专业约束 | 中 | 中-高 | NL/docs 与 executable 开始分叉 |
| 选型-布置-调度联合优化 | 中-高 | 高 | executable 显著提升 pass、repair 和质量 |

低耦合饱和任务不是反例。它们用于证明可执行约束在简单任务上更多体现为稳定性和可审计性。高耦合任务才是约束鸿沟的主要证据区。

## 关键对照

实验设计应避免把“上下文多寡”和“可执行反馈”混在一起。至少需要以下对照：

- 同一任务、同一 Agent、同一 scaffold，在 NL-only、Docs-only、Partial executable、Full executable 下比较。
- 同一 critic 结果，在不同 diagnostic 粒度下比较 repair。
- 对所有约束做完整评分，统计 Agent 在不可见约束和未反馈约束上的真实遗漏率。
- 对低耦合与高耦合任务分别报告结果，不只给一个总体平均值。

## 预期章节图表

- 覆盖率-质量曲线：x 轴为可执行约束覆盖率，y 轴为 pass rate、omission density 或 final score。
- 诊断粒度-repair 曲线：x 轴为 repair round，y 轴为 pass rate，按 diagnostic 条件分组。
- 任务复杂度分层表：低耦合、中耦合、高耦合任务分别报告 NL/docs/executable 差异。
- Top omission constraints：按约束族统计自然语言条件下最常漏掉的要求。
- Pseudo-correctness case studies：展示看起来合理但可执行检查失败的方案。
